# ProcureGen AI — 技术文档

> **采购文件智能生成系统**  
> 技术栈：Vue 3 + FastAPI + DeepSeek 大模型 + Jinja2 模板引擎 + python-docx  
> 日期：2026-07-02

---

## 一、环境要求

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| Python | 3.10+（推荐 3.11） | 通过 Conda 管理 |
| Node.js | 18+ | 前端构建工具链 |
| Conda | 任意版本 | 虚拟环境管理 |
| DeepSeek API Key | — | 若无，系统自动降级为 Mock 模式 |

## 二、快速开始

### 2.1 克隆 / 进入项目

```bash
cd procuregen-ai
```

### 2.2 后端环境配置

```bash
# 1. 创建 conda 虚拟环境
conda create -n procuregen python=3.11 -y

# 2. 激活环境
conda activate procuregen

# 3. 安装 Python 依赖
cd backend
pip install -r requirements.txt

# 4. 配置 DeepSeek API Key
# 编辑 backend/.env 文件，填入你的 API Key：
#   DEEPSEEK_API_KEY=sk-your-real-key-here
#   LLM_MODE=deepseek
#
# 如果没有 Key，改为：
#   LLM_MODE=mock
```

### 2.3 启动后端

```bash
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后：
- API 文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/

### 2.4 前端环境配置

```bash
cd frontend
npm install
```

### 2.5 启动前端

```bash
npm run dev
```

浏览器访问：http://localhost:5173

### 架构设计要点

| 设计点 | 说明 |
|--------|------|
| **LLM Adapter 抽象层** | 统一封装大模型调用，支持 DeepSeek / OpenAI-compatible / Mock 三种模式，通过 `.env` 文件的 `LLM_MODE` 一键切换 |
| **规则引擎 + 大模型双轨** | 风险检查优先走规则引擎（硬规则，零延迟），大模型只负责生成风险的自然语言解释。模型挂了也不影响风险检出 |
| **模板引擎控格式** | 所有输出文档先经过 Jinja2 模板渲染，大模型只输出结构化 JSON，绝不直接生成文档正文，避免格式不稳定 |
| **Mock 降级保底** | 无 API Key 时自动使用内置样例数据返回，保证课程演示闭环 |
| **步骤式前端流程** | 不拆多个路由页面，一个 `GenerateView.vue` + `el-steps` 步骤条串联全流程，降低前端复杂度 |

## 五、API 接口文档

所有接口统一返回格式：

```json
{
  "success": true/false,
  "data": { ... },
  "message": "说明信息"
}
```

| 方法 | 路径 | 功能 | 关键参数 |
|------|------|------|---------|
| POST | `/api/parse` | 解析采购需求 | `input_text`（必填）, `mode`（可选：llm/mock） |
| POST | `/api/risk-check` | 执行合规风险预审 | `parsed_spec`（必填）, `selected_clauses`（可选） |
| POST | `/api/generate` | 生成 Word 文档 | `parsed_spec`（必填）, `template_id`（可选）, `selected_clauses`（可选） |
| GET | `/api/tasks` | 获取历史任务列表 | `page`, `size` |
| GET | `/api/tasks/{id}` | 获取任务详情 | `id` |
| GET | `/api/export/{id}` | 下载 Word 文件 | `id` |
| GET | `/api/templates` | 获取模板列表 | `template_type`（可选） |
| POST | `/api/templates` | 新增模板 | `name`, `template_type`, `content` |
| GET | `/api/clauses` | 获取条款列表 | `category`（可选） |
| POST | `/api/clauses` | 新增条款 | `title`, `category`, `content`, `applicable_condition` |

## 六、数据库设计

使用 SQLite，数据库文件自动创建在 `backend/procuregen.db`。

### 6.1 表结构

| 表名 | 用途 |
|------|------|
| `templates` | 招标书/合同模板（Jinja2 格式） |
| `clauses` | 可复用合同条款库 |
| `risk_rules` | 合规风险检查规则 |
| `generation_tasks` | 每次生成任务记录 |

### 6.2 关键字段

```
templates: id, name, template_type, content(Jinja2), variables, created_at

clauses: id, title, category, content, applicable_condition, created_at

risk_rules: id, rule_name, rule_type, severity(high/medium/low),
            condition_expr, message_template, suggestion, created_at

generation_tasks: id, input_text, parsed_spec(JSON), selected_clauses(JSON),
                  risk_report(JSON), file_path, file_name, status, created_at
```

### 6.3 规则条件表达式

规则引擎支持两种条件格式：

| 表达式 | 含义 |
|--------|------|
| `is_empty:budget` | budget 字段为空时触发 |
| `is_empty:payment_terms\|acceptance_criteria` | 多个字段任一为空时触发 |
| `contains:qualification:品牌` | qualification 数组包含"品牌"关键词时触发 |

## 七、大模型集成说明

### 7.1 DeepSeek API

系统通过 OpenAI-compatible 接口调用 DeepSeek：

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-key",
    base_url="https://api.deepseek.com/v1",
)
response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[...],
    temperature=0.1,
    max_tokens=4096,
)
```

### 7.2 Mock 模式

当 `LLM_MODE=mock` 或无 API Key 时，系统返回内置样例数据：

- 解析接口返回"人工智能实验室服务器采购项目"的完整 JSON
- 风险解释由规则引擎直接生成，不调用大模型
- 所有功能均可正常演示

### 7.3 防幻觉设计

1. 模型只输出 JSON，后端用 Pydantic 校验
2. 文档正文由 Jinja2 模板控制，模型不直接生成文档
3. 规则库中的法规提示由管理员维护，模型不做"法律结论"
4. 用户确认步骤（Step 2 → Step 3）保证人工把关

## 八、前端页面说明

### 8.1 页面路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | HomeView | 首页工作台，项目介绍 + 快捷入口 + 最近记录 |
| `/generate` | GenerateView | 生成流程（步骤条串联 4 个 Step） |
| `/history` | HistoryView | 历史记录列表，查看详情和下载 |

### 8.2 生成流程步骤

```
Step 0: 输入需求  →  大文本框输入自然语言描述
Step 1: 解析结果  →  展示 AI 抽取的结构化字段，允许手动编辑
Step 2: 风险报告  →  展示风险等级、风险项及修改建议
Step 3: 预览下载  →  展示推荐条款、文档预览、下载 Word
```

## 十一、常见问题

**Q: 没有 DeepSeek API Key 能演示吗？**
A: 能。将 `backend/.env` 中的 `LLM_MODE` 改为 `mock`，系统使用内置样例数据。

**Q: 如何获取 DeepSeek API Key？**
A: 访问 https://platform.deepseek.com 注册并获取。

**Q: 端口被占用怎么办？**
A: 后端端口在启动命令中修改 `--port`，前端端口在 `vite.config.js` 中修改 `server.port`。

**Q: 数据库在哪里？**
A: 首次启动后端时自动在 `backend/procuregen.db` 创建，同时自动填充种子数据。

**Q: 导出的 Word 文件在哪里？**
A: 在 `exports/` 目录下，文件名格式：`项目名称_随机ID.docx`。
